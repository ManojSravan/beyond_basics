# beyond_basics

As a product engineer, I constantly found myself context-switching across codebases, tools, and concepts I *know* but need a second to recall. Googling the same things repeatedly, skimming docs for that one pattern, re-reading articles I've read before.

So I built this instead.

**beyond_basics** is a structured, no-fluff reference for engineers who already know the fundamentals and want one reliable place to look things up. Not a tutorial, not a course. Just clean, dense, useful cheatsheets written the way I'd want to read them.

This is also open source. If you're learning in public, think something's missing, or want to improve what's here — contributions are welcome.

## How it's structured

Every cheatsheet follows the same format so you always know what to expect.

- **Tables** for quick pattern lookup — what it is, when to use it, example
- **Code examples** that are real and copy-paste useful, not toy snippets
- **Tradeoffs** so you understand *why* to pick something, not just *how* to use it
- **Anti-patterns** so you know what to avoid and why
- **Quick reference** at the end of every sheet for fast scanning

## Contributing

This repo follows a simple structure and that's intentional. If you want to contribute, keep the same format.

- Each cheatsheet lives in its relevant folder
- Follow the existing table and section structure
- Code examples should be practical, not minimal toy examples
- Include tradeoffs and anti-patterns where relevant
- Open a PR with a short description of what you're adding or improving

If you want to add a new topic, open an issue first so we can align on scope and structure before you put in the effort.

Have a topic you'd like covered? Open an issue.
