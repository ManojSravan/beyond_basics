# React Data Fetching Cheatsheet

---

## 1. Mental Model — Server State vs Client State

> Server state and client state are fundamentally different problems. Don't manage them the same way.

| | Client State | Server State |
|---|---|---|
| **Lives in** | Memory (your app) | Database / API |
| **Owned by** | You | The server |
| **Examples** | Modal open, selected tab, form input | Users, products, orders |
| **Challenge** | Predictability | Caching, syncing, staleness |
| **Tool** | useState / Zustand | TanStack Query / SWR |

## 2. Fetching with `useEffect` (baseline pattern)

The raw approach — useful to understand before abstractions.

```tsx
function ProductList() {
  const [products, setProducts] = useState([])
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState(null)

  useEffect(() => {
    let cancelled = false // prevent state update on unmounted component

    async function fetch() {
      try {
        setLoading(true)
        const res = await fetch('/api/products')
        if (!res.ok) throw new Error('Failed to fetch')
        const data = await res.json()
        if (!cancelled) setProducts(data)
      } catch (err) {
        if (!cancelled) setError(err)
      } finally {
        if (!cancelled) setLoading(false)
      }
    }

    fetch()
    return () => { cancelled = true } // cleanup — avoids race conditions
  }, [])

  if (loading) return <Spinner />
  if (error) return <ErrorMessage error={error} />
  return <ul>{products.map(p => <li key={p.id}>{p.name}</li>)}</ul>
}
```

### Why this gets painful at scale

| Problem | What happens |
|---|---|
| No caching | Every mount re-fetches, even same data |
| No deduplication | Multiple components trigger same request simultaneously |
| No background sync | Data goes stale silently |
| No loading/error states | You write them manually every time |
| Race conditions | Slow response arrives after faster one |
| Refetch on focus | Doesn't happen — user sees stale data |

## 3. Custom Fetch Hook (intermediate pattern)

Extract the boilerplate into a reusable hook.

```tsx
function useFetch<T>(url: string) {
  const [data, setData] = useState<T | null>(null)
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState<Error | null>(null)

  useEffect(() => {
    let cancelled = false

    setLoading(true)
    fetch(url)
      .then(res => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`)
        return res.json()
      })
      .then(data => { if (!cancelled) setData(data) })
      .catch(err => { if (!cancelled) setError(err) })
      .finally(() => { if (!cancelled) setLoading(false) })

    return () => { cancelled = true }
  }, [url])

  return { data, loading, error }
}

// Usage
function ProductList() {
  const { data, loading, error } = useFetch<Product[]>('/api/products')
  // ...
}
```

> Still no caching or deduplication. Good for simple apps, not for anything serious.

## 4. TanStack Query (React Query) — recommended

**Install**
```bash
npm install @tanstack/react-query
```

**Setup**
```tsx
// main.tsx
import { QueryClient, QueryClientProvider } from '@tanstack/react-query'

const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 60 * 1000,      // data fresh for 1 min
      retry: 2,                   // retry failed requests twice
      refetchOnWindowFocus: true, // sync when user returns to tab
    }
  }
})

ReactDOM.createRoot(document.getElementById('root')!).render(
  <QueryClientProvider client={queryClient}>
    <App />
  </QueryClientProvider>
)
```

## 5. useQuery — Reading Data

```tsx
import { useQuery } from '@tanstack/react-query'

const fetchProducts = () => fetch('/api/products').then(r => r.json())

function ProductList() {
  const { data, isLoading, isError, error, isFetching, refetch } = useQuery({
    queryKey: ['products'],         // cache key — must be unique and descriptive
    queryFn: fetchProducts,
    staleTime: 5 * 60 * 1000,      // override default per-query
  })

  if (isLoading) return <Spinner />    // true only on first load
  if (isError) return <Error msg={error.message} />

  return (
    <>
      {isFetching && <SmallSpinner />} {/* background refetch indicator */}
      <ul>{data.map(p => <li key={p.id}>{p.name}</li>)}</ul>
      <button onClick={() => refetch()}>Refresh</button>
    </>
  )
}
```

### Query Key Patterns — most important concept in React Query

```tsx
// Static key — no params
useQuery({ queryKey: ['products'], queryFn: fetchProducts })

// Dynamic key — refetches when id changes
useQuery({ queryKey: ['products', id], queryFn: () => fetchProduct(id) })

// With filters — refetches when filters change
useQuery({ queryKey: ['products', { category, page, sort }], queryFn: () => fetchProducts({ category, page, sort }) })

// Key as dependency — treat it like useEffect deps array
// If it's in the queryFn, it must be in the queryKey
```

### `isLoading` vs `isFetching`

| | `isLoading` | `isFetching` |
|---|---|---|
| **True when** | First load, no cached data | Any fetch in progress (including background) |
| **Use for** | Full page skeleton/spinner | Subtle loading indicator |

## 6. useMutation — Writing Data

```tsx
import { useMutation, useQueryClient } from '@tanstack/react-query'

function AddProduct() {
  const queryClient = useQueryClient()

  const mutation = useMutation({
    mutationFn: (newProduct) =>
      fetch('/api/products', {
        method: 'POST',
        body: JSON.stringify(newProduct),
        headers: { 'Content-Type': 'application/json' }
      }).then(r => r.json()),

    onSuccess: () => {
      // Invalidate cache — triggers refetch of product list
      queryClient.invalidateQueries({ queryKey: ['products'] })
    },

    onError: (error) => {
      console.error('Failed to add product:', error)
    }
  })

  return (
    <button
      onClick={() => mutation.mutate({ name: 'New Product', price: 99 })}
      disabled={mutation.isPending}
    >
      {mutation.isPending ? 'Adding...' : 'Add Product'}
    </button>
  )
}
```

## 7. Optimistic Updates

Update the UI immediately, roll back if the server fails.

```tsx
const mutation = useMutation({
  mutationFn: (updatedProduct) =>
    fetch(`/api/products/${updatedProduct.id}`, {
      method: 'PUT',
      body: JSON.stringify(updatedProduct)
    }),

  onMutate: async (updatedProduct) => {
    // 1. Cancel outgoing refetches (avoid overwriting optimistic update)
    await queryClient.cancelQueries({ queryKey: ['products'] })

    // 2. Save current cache value (for rollback)
    const previous = queryClient.getQueryData(['products'])

    // 3. Optimistically update cache
    queryClient.setQueryData(['products'], (old: Product[]) =>
      old.map(p => p.id === updatedProduct.id ? updatedProduct : p)
    )

    // 4. Return snapshot for rollback
    return { previous }
  },

  onError: (err, _, context) => {
    // Rollback on failure
    queryClient.setQueryData(['products'], context?.previous)
  },

  onSettled: () => {
    // Always sync with server after mutation
    queryClient.invalidateQueries({ queryKey: ['products'] })
  }
})
```

## 8. Dependent Queries

Fetch B only when A has resolved.

```tsx
function UserOrders({ userId }: { userId: string }) {
  // Step 1 — fetch user
  const { data: user } = useQuery({
    queryKey: ['user', userId],
    queryFn: () => fetchUser(userId)
  })

  // Step 2 — fetch orders only when user is loaded
  const { data: orders } = useQuery({
    queryKey: ['orders', user?.accountId],
    queryFn: () => fetchOrders(user!.accountId),
    enabled: !!user?.accountId, // key — disables query until condition is true
  })

  return <OrderList orders={orders} />
}
```

## 9. Parallel Queries

```tsx
// Multiple independent queries — fire simultaneously
function Dashboard() {
  const productsQuery = useQuery({ queryKey: ['products'], queryFn: fetchProducts })
  const ordersQuery   = useQuery({ queryKey: ['orders'],   queryFn: fetchOrders })
  const usersQuery    = useQuery({ queryKey: ['users'],    queryFn: fetchUsers })

  const isLoading = productsQuery.isLoading || ordersQuery.isLoading || usersQuery.isLoading

  // ...
}

// Dynamic parallel queries with useQueries
function MultiProductDetail({ ids }: { ids: string[] }) {
  const queries = useQueries({
    queries: ids.map(id => ({
      queryKey: ['product', id],
      queryFn: () => fetchProduct(id)
    }))
  })

  return queries.map((q, i) => <ProductCard key={ids[i]} product={q.data} />)
}
```

## 10. Pagination

```tsx
function PaginatedProducts() {
  const [page, setPage] = useState(1)

  const { data, isPlaceholderData } = useQuery({
    queryKey: ['products', page],
    queryFn: () => fetchProducts({ page }),
    placeholderData: keepPreviousData, // keeps old page visible while new one loads
  })

  return (
    <>
      <ProductList products={data?.items} />
      <div>
        <button onClick={() => setPage(p => p - 1)} disabled={page === 1}>Prev</button>
        <span>Page {page}</span>
        <button
          onClick={() => setPage(p => p + 1)}
          disabled={isPlaceholderData || !data?.hasMore}
        >
          Next
        </button>
      </div>
    </>
  )
}
```

### Infinite Scroll

```tsx
import { useInfiniteQuery } from '@tanstack/react-query'

function InfiniteProductList() {
  const { data, fetchNextPage, hasNextPage, isFetchingNextPage } = useInfiniteQuery({
    queryKey: ['products', 'infinite'],
    queryFn: ({ pageParam }) => fetchProducts({ cursor: pageParam }),
    initialPageParam: null,
    getNextPageParam: (lastPage) => lastPage.nextCursor ?? undefined,
  })

  const allProducts = data?.pages.flatMap(page => page.items) ?? []

  return (
    <>
      {allProducts.map(p => <ProductCard key={p.id} product={p} />)}
      <button
        onClick={() => fetchNextPage()}
        disabled={!hasNextPage || isFetchingNextPage}
      >
        {isFetchingNextPage ? 'Loading...' : hasNextPage ? 'Load more' : 'No more'}
      </button>
    </>
  )
}
```

## 11. Prefetching

Load data before the user navigates to it.

```tsx
// Prefetch on hover — data ready before user clicks
function ProductLink({ id }: { id: string }) {
  const queryClient = useQueryClient()

  return (
    <Link
      to={`/products/${id}`}
      onMouseEnter={() =>
        queryClient.prefetchQuery({
          queryKey: ['product', id],
          queryFn: () => fetchProduct(id),
          staleTime: 10 * 1000, // only prefetch if data older than 10s
        })
      }
    >
      View Product
    </Link>
  )
}

// Prefetch in route loader (TanStack Router / React Router v6.4+)
export async function loader({ params }) {
  await queryClient.prefetchQuery({
    queryKey: ['product', params.id],
    queryFn: () => fetchProduct(params.id)
  })
  return null
}
```

## 12. Error Handling Patterns

```tsx
// Per-query error handling
const { data, isError, error } = useQuery({ ... })
if (isError) return <ErrorMessage message={error.message} />

// Global error handler
const queryClient = new QueryClient({
  queryCache: new QueryCache({
    onError: (error, query) => {
      if (query.state.data !== undefined) {
        // Only show toast for background refetch errors (data already cached)
        toast.error(`Sync failed: ${error.message}`)
      }
    }
  }),
  mutationCache: new MutationCache({
    onError: (error) => toast.error(error.message)
  })
})
```

## 13. SWR (alternative to React Query)

**Lighter, simpler, less features.** Good for basic cases.

```bash
npm install swr
```

```tsx
import useSWR from 'swr'

const fetcher = (url: string) => fetch(url).then(r => r.json())

function ProductList() {
  const { data, error, isLoading, mutate } = useSWR('/api/products', fetcher, {
    refreshInterval: 30000,       // poll every 30s
    revalidateOnFocus: true,      // refetch on tab focus
    dedupingInterval: 2000,       // dedup requests within 2s
  })

  if (isLoading) return <Spinner />
  if (error) return <Error />
  return <ul>{data.map(p => <li key={p.id}>{p.name}</li>)}</ul>
}
```

| | TanStack Query | SWR |
|---|---|---|
| **Bundle size** | ~13KB | ~4KB |
| **Mutations** | Full support with `useMutation` | Manual with `mutate()` |
| **Devtools** | Excellent | Basic |
| **Infinite queries** | Built-in | Built-in |
| **Optimistic updates** | First-class | Manual |
| **Best for** | Complex apps | Simple to medium apps |

## 14. Race Condition Patterns

```tsx
// ❌ Classic race condition — if user types fast, old response may arrive last
useEffect(() => {
  fetch(`/api/search?q=${query}`).then(r => r.json()).then(setResults)
}, [query])

// ✅ Fix 1 — cancellation flag
useEffect(() => {
  let cancelled = false
  fetch(`/api/search?q=${query}`)
    .then(r => r.json())
    .then(data => { if (!cancelled) setResults(data) })
  return () => { cancelled = true }
}, [query])

// ✅ Fix 2 — AbortController (also cancels the network request)
useEffect(() => {
  const controller = new AbortController()
  fetch(`/api/search?q=${query}`, { signal: controller.signal })
    .then(r => r.json())
    .then(setResults)
    .catch(err => { if (err.name !== 'AbortError') setError(err) })
  return () => controller.abort()
}, [query])

// ✅ Fix 3 — just use TanStack Query, it handles this for you
useQuery({ queryKey: ['search', query], queryFn: () => searchProducts(query) })
```

## 15. Quick Reference

```
Fetching strategy decision:

Simple component, no sharing needed?
  └── useFetch custom hook

Shared data, caching matters, multiple components?
  └── TanStack Query (useQuery)

Writing data back to server?
  └── useMutation + invalidateQueries

Need instant UI feedback?
  └── Optimistic updates with onMutate

Long lists or feeds?
  └── useInfiniteQuery

Lightweight app, don't need devtools?
  └── SWR

staleTime guide:
  0          → always refetch (real-time data)
  30s–2min   → dashboards, feeds
  5min       → general app data
  Infinity   → static/reference data (config, lookup tables)
```
